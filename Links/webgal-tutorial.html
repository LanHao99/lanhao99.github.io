第一章

脚本基础与对话编写

欢迎来到WebGal脚本教程的第一章。本章将带你从零开始，掌握WebGal脚本的基础语法和对话编写方法，为你的视觉小说创作之旅打下坚实基础。

1 脚本基础结构

start.txt 入口文件

WebGAL的场景是以文件为单位的，后缀名为.txt的脚本文件。就像很多编程语言有一个main函数作为入口一样，WebGAL的入口场景是start.txt。

重要说明

  • 程序启动时会从初始脚本 start.txt 开始运行，请不要重命名或者删除
  • WebGAL会首先尝试获取start.txt，然后调用解析器解析为引擎可执行的场景对象
  • 使用choose、changeScene、callScene可以切换或"调用"场景

注释写法

WebGAL脚本只会解析每一行的分号前的内容，因此分号后的内容会被视作注释。

// 单行注释示例
WebGAL:你好！; 分号后的内容会被视作注释
; 可以直接输入一个分号，然后写一条单行注释

脚本基本语法

基本结构由指令、内容和参数组成。

command:content -arg1 -arg2 ...;comment
command: 语句指令 (如bgm)
content: 语句主要内容
-arg: 附加参数

语句解析规则

在content后以 - 分隔的是附加参数。

注意

附加参数的连字符-前需要有空格，否则WebGAL可能会认为这不是一个参数。

参数以 -key=value 形式表示：

  • • string: -key=s
  • • number: -key=1
  • • boolean: -key=true

2 关键参数详解

none 关键词

在设置立绘、BGM、背景等资源时，通过设置为 none，可以关闭这个对象。

changeFigure:none; // 关闭立绘
bgm:none; // 关闭背景音乐
changeBg:none; // 关闭背景

-next 参数

在任意语句后加上参数 -next，可以在执行完本条语句后立刻跳转到下一条语句。

语法糖：只写出key而省略value会被解析为 -key=true

changeBg:testBG03.jpg -next; // 等同于 -next=true

3 角色对话编写

基本角色对话

编写角色对话的方式非常简单：输入 角色:对话;。

基本规则：

  • 冒号应为英文字符
  • 使用分号作为结束
  • WebGAL解析为 say:内容 -speaker=角色
雪之下雪乃:请用茶;
由比滨:啊，谢谢;
小町:谢谢雪乃姐！;

连续对话

如果多条对话没有改变角色名称，可以省略角色名称。

雪之下雪乃:你到得真早;
// 此时，对话的角色名称仍然是 "雪之下雪乃"
对不起，等很久了吗？;
比企谷八幡:刚到而已;
优势：减少重复输入，提高编写效率

对话中的多行文字

可以使用竖线(|)来分割多行文字。

角色A:第一行文字|第二行文字|第三行文字;

4 旁白与黑屏文字

旁白

没有角色名称的对话，用于描述场景、心理活动。请在冒号前留空（但不能省去冒号）。

:这是一句旁白;

解析为：say:这是一句旁白 -clear;

黑屏文字

以黑屏显示文字，用于引入主题或表现心理活动。使用 intro 命令。

intro:回忆不需要适合的剧本，|反正一说出口，|都成了戏言。;
使用 -hold 参数可在独白结束后保持界面
特性 黑屏文字 旁白
命令 intro say (带-clear)
界面 黑屏全屏 对话框
背景显示 无 可有
用途 开场、过渡、独白 场景描述、动作描写

7 常见问题

脚本不执行或报错

原因：缺少分号、使用了中文冒号或分号、文件编码问题。

确保所有标点使用英文字符，每条语句以分号结束，使用UTF-8编码保存。

参数不生效

原因：连字符前缺少空格、参数名拼写错误。

主角:你好-speaker=小明; 主角:你好 -speaker=小明;

第二章

背景与立绘视觉控制

在WebGal视觉小说制作中，背景和立绘是构建游戏画面的核心元素。通过简单的脚本命令，你可以轻松控制背景的切换、人物立绘的显示与隐藏，以及立绘在屏幕上的位置和动画效果。

背景与立绘基础控制

资源准备

在开始之前，请确保资源文件放置在正确的文件夹中：

  • background 文件夹：存放背景图片
  • figure 文件夹：存放立绘图片[95]

立即执行参数 (-next)

通常改变背景/立绘后需要点击才能显示下一条对话。使用 -next 参数可立即执行下一条语句。

Example 
changeBg:testBG03.jpg -next;
changeFigure:testFigure02.png -next;
一色:谢谢学姐！;

改变背景图片

使用 changeBg 命令来切换当前显示的背景图片。

WebGal Script 
changeBg:testBG03.jpg; // 改变背景
changeBg:none; // 关闭背景

改变人物立绘

使用 changeFigure 命令来切换立绘，并支持位置参数控制。

Position Control 
changeFigure:testFigure03.png -left;  // 左侧
changeFigure:testFigure04.png;         // 中间（默认）
changeFigure:testFigure03.png -right; // 右侧
立绘ID控制

为立绘指定 -id 可实现更精确的控制，支持超过3个立绘的使用场景。

多段动画实现

使用 setTempAnimation 实现复杂的连续动作

通过定义多个动画片段，可以实现从A到B再到C的连续运动。每个片段都可以设置独立的持续时间和缓动效果。

  • 支持多种缓动类型
  • 可叠加位置、旋转、缩放等属性
  • 灵活控制动画节奏 
// 角色进场动画
changeFigure:heroine.png -id=heroine1 
  -transform={"position":{"x":-600,"y":0}};

// 多段动画：从左侧滑入
setTempAnimation:[
  {"position":{"x":0},"duration":800,"ease":"easeOut"}
] -target=heroine1;

小头像与对话框控制

miniAvatar 命令

小头像通常显示在对话框旁，用于标识当前说话的角色。miniAvatar 是一个三合一命令，集显示、改变、隐藏功能于一体[223]。

1

显示角色头像

miniAvatar:character_a/avatar.png;

2

切换情绪表情

3

样式建议
  • • 尺寸保持一致

  • • 突出表情特征

性能优化与注意事项

优化建议
  • 合理使用 -next 参数避免冗余点击
  • 动画时长控制在 200-1000ms 范围内
  • 确保资源在需要前已预加载完成
  • 及时关闭不需要的立绘和效果
关键注意

  • 动画第一片段通常 duration 设为 0
  • CG 图片路径相对于项目根目录
  • 重置带ID立绘位置需先关闭再打开
第三章

场景管理与剧情分支

场景管理与剧情分支是WebGAL脚本的核心功能。通过合理运用场景切换、调用、标签跳转和分支选择指令，构建丰富多样的互动体验，是开发高质量视觉小说的关键。

场景系统基础

WebGAL的场景系统是基于文件的，通常为带有 .txt 后缀的脚本文件。引擎的入口场景固定为 start.txt。解析器会将这些文件转换为可执行的场景对象，并自动预加载所需的音频、视频和图像资源[45]。

场景切换

使用 changeScene 指令永久性地跳转到另一个场景文件，替换当前场景。

语法示例
# 直接切换，无法返回
changeScene:chapter_school.txt;
适用于章节推进、游戏结束等无需返回的场景。

场景调用

使用 callScene 将新场景推入调用栈，执行完毕后自动返回父场景。

语法示例
# 调用子场景，执行完后返回
callScene:shop.txt;
适用于商店、设置、背包等临时界面。

场景调用栈机制

当使用 callScene 时，当前场景的状态会被保存（压栈）。子场景执行完毕后，引擎从栈顶弹出父场景并恢复执行（出栈）。这与 changeScene 直接替换且不保留状态有本质区别[45]。

特性 callScene changeScene
返回能力 可以返回 无法返回
状态保留 保留父场景状态 不保留
调用栈 推入栈 不涉及栈

标签跳转

在同一场景文件内，使用 label 标记位置，使用 jumpLabel 跳转到指定位置。这是实现场景内分支和循环的关键机制。

基本用法

示例
choose:分支1:label_1|分支2:label_2;

label:label_1;
dialogue:主角:选择了分支1。;
jumpLabel:end;

label:label_2;
dialogue:主角:选择了分支2。;
jumpLabel:end;

label:end;
dialogue:旁白:流程结束。;

最佳实践

  • 统一汇聚点：设置如 label:end; 的统一结束点，避免代码重复执行[26]。
  • 性能考量：避免在超长分支中使用标签跳转，建议拆分为独立场景文件以防加载缓慢[26]。
  • 命名规范：使用有意义的名称，如 label:start_intro;。

⚠️ 重要：防止"穿透"执行

WebGAL脚本是从上到下线性执行的。如果你在分支结尾忘记使用 jumpLabel 跳转，程序会继续向下执行下一个分支的内容[26]。务必在每个分支逻辑结束时跳转到汇聚点。

分支选择

choose 指令不仅提供基本的选项跳转，还支持强大的条件判断功能。

基本语法

choose:选项文本:目标文件|选项文本:目标文件;

条件显示与启用

使用 (展示条件) 控制是否显示，使用 [启用条件] 控制是否可点击[48]。 

# 只有好感度>=10才显示
# 只有金币>=100才能点击
choose:(love>=10)[money>=100]->购买药水:buy.txt|离开:leave.txt;

分支选择界面示例

资源预加载机制

WebGAL解析器会自动扫描当前场景及其调用的子场景（如 choose 指向的文件），合并所有需要的资源并提前加载[45]。

  • 识别 bgm 音频资源
  • 识别 playVideo 视频资源
  • 识别 changeBg 图像资源

提升场景切换流畅度，减少等待时间。

场景切换流程

引擎通过预步骤操作（停止动画、检查状态）确保平滑过渡[45]。

💡 综合实战：好感度分支系统

结合变量、条件判断和场景调用的完整示例。

game_start.txt WebGAL Script 
# 初始化变量
setVar:love=0;
setVar:confidence=0;

dialogue:主角:今天是开学的第一天。;
choose:主动搭讪:approach.txt|默默观察:observe.txt;

# --- approach.txt ---
dialogue:主角:你好，我可以和你做朋友吗？;
setVar:love=love+5;  # 增加好感度
setVar:confidence=confidence+2;

# 根据好感度决定后续分支
# 只有 love>10 且 confidence>5 时显示"主动和她一组"选项
choose:(love>10)[confidence>5]->主动与她一组:group_with_her.txt|等待分配:wait.txt;

常见问题 (FAQ)

Q1 为什么我的分支会继续执行其他分支的代码？

原因：忘记在分支结尾使用 jumpLabel 跳转。解决：在每个分支结尾添加跳转到汇聚点的语句[26]。

Q2 什么时候使用 callScene，什么时候使用 changeScene？

临时场景（商店、设置、背包）使用 callScene，以便返回；章节切换、结局跳转等永久性切换使用 changeScene。

Q3 如何实现条件选项（如只有有钱才能买）？

使用条件语法：choose:(展示条件)[启用条件]->选项文本:目标;。例如 (money>=100)[hasKey==true]->购买道具:buy.txt[48]。

第四章

变量系统与交互逻辑

变量系统是WebGal脚本的核心功能，它赋予游戏动态性和交互能力。通过变量，你可以存储游戏状态、实现条件判断、处理用户输入，构建复杂的游戏逻辑。

变量定义与基础操作

setVar 指令

使用 setVar 指令来设置变量，这是操作变量的核心命令。

setVar:a=1;
setVar:name="玩家";
setVar:isValid=true;

数据类型

WebGal支持三种基本数据类型，涵盖数字、逻辑判断和文本内容。

  • Number 整数和浮点数 (42, 3.14)
  • Boolean true 或 false
  • String 文本内容 ("Hello")

变量作用域

根据生命周期的不同，变量分为运行时、存档和全局三种。

运行时变量 场景结束自动释放
存档变量 跟随存档保存
全局变量 跨周目持久化 (-global)

随机数生成

音频与视频多媒体集成

多媒体资源是视觉小说游戏的灵魂。本章将系统性指导你在 WebGal 脚本中集成各类多媒体资源，掌握 BGM、角色语音、SFX 及视频的播放与高级控制技巧，营造沉浸式游戏体验。

BGM 背景音乐集成

背景音乐是贯穿场景的主要音频元素。使用 bgm 指令即可轻松播放位于 bgm 文件夹中的音频文件。

基础语法

bgm:文件名.扩展名;

音量控制

使用 -volume 参数调整音量（0-100）。

Example
bgm:夏影.mp3 -volume=30;

淡入淡出

使用 -enter 设置淡入时间（毫秒）。

Example
bgm:夏影.mp3 -enter=3000;

角色语音集成

在对话脚本中加入 -语音文件名 参数即可同步播放语音。文件需存放在 vocal 文件夹。 

比企谷八幡:刚到而已 -V3.ogg -volume=80;
连续对话时，WebGal 会自动管理语音播放顺序，确保每句对话的语音在其显示时开始播放。

效果音 (SFX)

使用 playEffect 指令播放环境音效（如开门、雨声）。通过 -id 参数可实现循环播放。 

playEffect:rain.mp3 -id=rain -volume=20;

视频播放集成

视频常用于片头（OP）、片尾（ED）或关键剧情过场。使用 playVideo 指令播放位于 video 文件夹中的视频。

禁止跳过参数

使用 -skipOff 参数可强制用户观看完整视频，适用于重要的剧情过场。

Example
playVideo:OP.mp4 -skipOff;

资源预加载机制

WebGal的特效系统基于PixiJS实现，提供了强大的图形渲染能力，支持硬件加速的雨雪等天气效果[163]。

自定义特效开发

通过编写JavaScript代码创建 PIXI.Container 对象，可以实现萤火虫、粒子流等复杂效果[163]。

fireflies.js 
const { registerEffect } = require('./effectRegistry');

registerEffect('fireflies', () => {
  const container = new PIXI.Container();
  
  // 创建20只萤火虫
  for (let i = 0; i < 20; i++) {
    const firefly = new PIXI.Graphics();
    firefly.beginFill(0xFFFF00, 0.8);
    firefly.drawCircle(0, 0, 2);
    firefly.endFill();
    firefly.x = Math.random() * window.innerWidth;
    firefly.y = Math.random() * window.innerHeight;
    
    // 设置随机速度和方向
    const speed = Math.random() * 0.5 + 0.2;
    const angle = Math.random() * Math.PI * 2;
    firefly.vx = Math.cos(angle) * speed;
    firefly.vy = Math.sin(angle) * speed;
    
    // 更新位置函数
    firefly.update = () => {
      firefly.x += firefly.vx;
      firefly.y += firefly.vy;
      
      // 边界循环
      if (firefly.x < 0) firefly.x = window.innerWidth;
      if (firefly.x > window.innerWidth) firefly.x = 0;
      if (firefly.y < 0) firefly.y = window.innerHeight;
      if (firefly.y > window.innerHeight) firefly.y = 0;
    };
    
    container.addChild(firefly);
  }
  
  // 动画循环
  const animate = () => {
    container.children.forEach(f => f.update());
    requestAnimationFrame(animate);
  };
  animate();
  
  return container;
});
